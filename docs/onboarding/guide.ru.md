## 2. Типы счетов в Sure

Sure поддерживает несколько типов счетов, которые делятся на **Активы** (то, чем вы владеете) и **Долги/Обязательства** (то, что вы должны):

| Активы      | Долги/Обязательства |
| ----------- | ------------------- |
| Наличные    | Кредитная карта     |
| Инвестиции  | Заем                |
| Крипто      | Прочие обязательства |
| Недвижимость |                     |
| Транспорт   |                     |
| Другой актив |                     |


## 3. Как работают счета активов

Наличные, расчетные и сберегательные счета **увеличиваются**, когда вы добавляете деньги, и **уменьшаются**, когда вы тратите деньги.

Пример:

- Начальный баланс: $500
- Добавили расход $20 → баланс стал $480
- Добавили доход $100 → баланс стал $580


## 4. Как работают долговые счета (обязательства)

Счета обязательств показывают, сколько денег вы **должны**, поэтому математика может показаться *обратной* по сравнению со счетом актива.

**Ключевое правило:**

- **Положительный баланс** = вы должны деньги
- **Отрицательный баланс** = банк должен *вам* (например, переплата или возврат)

**Транзакции ведут себя так:**

- **Расходы** (например, покупки) → увеличивают долг (вы должны больше)
- **Платежи или возвраты** → уменьшают долг (вы должны меньше)

Пример для кредитной карты:

1. Баланс: **долг $200**
2. Потратили $20 → теперь долг $220 (баланс *растет*, красный)
3. Оплатили $50 → теперь долг $170 (баланс *уменьшается*, зеленый)

Пример переплаты:

1. Баланс: -$44 (банк должен вам $44)
2. Потратили $1 → банк теперь должен **$43** (баланс отображается как -$43 и движется к нулю)

> [!TIP]
> Почему так? Это соответствует стандартному учету и тому, как банки показывают баланс в личных кабинетах. Думайте о балансе обязательств как о «**Сумме долга**», а не как о «доступных средствах».


## 5. Быстрая памятка: активы vs. обязательства

| Действие         | Счет актива (например, расчетный) | Счет обязательства (например, кредитка) |
| ---------------- | --------------------------------- | ---------------------------------------- |
| Потратить $20    | Баланс ↓ $20                      | Баланс ↑ $20 (долг больше)              |
| Получить $50     | Баланс ↑ $50                      | Баланс ↓ $50 (долг меньше)              |
| Отрицательный баланс | Овердрафт                     | Банк должен *вам* деньги                |


## 6. Добавление счетов

Для примера добавим **сберегательный счет**.<br />

> [!TIP]
> Если добавляете **кредитную карту**, **заем** или другое **обязательство**, обязательно выбирайте тип **Кредитная карта** или **Обязательство**, а не **Наличные**. Это гарантирует корректный расчет балансов и соответствие данным банка.

Большинство банковских счетов (расчетные, сберегательные, денежный рынок) — это **счета наличных**.
1. Нажмите **+ Add Account** → **Cash** → **Enter Account Balance**
2. Заполните детали:
   - Название счета
   - Текущий баланс
   - Подтип счета (здесь указываются расчетный, сберегательный и т.д.)
3. Нажмите **Create Account**, чтобы завершить.

После создания вы вернетесь на **Главный** экран.<br />

- Новый счет наличных в списке **Счета** (слева)
- Обзор счетов в центре, под полосой чистой стоимости (net worth).

## 7. Добавление транзакций

Чтобы добавить транзакцию:
1. Перейдите в **Транзакции** (левое меню, под **Главной**, выше **Бюджетов**)
2. Нажмите **+ New Transaction** (вверху справа)
3. Выберите тип транзакции:
   - **Expense** → расход
   - **Income** → доход
   - **Transfer** → перевод между счетами
4. Введите детали и нажмите **Add transaction**

Теперь транзакция появится в **истории**, а **график чистой стоимости** обновится.

## 8. Управление инвестиционными счетами

Если вы ведете инвестиции в Sure, есть дополнительные функции для точного учета портфеля.

### Учет стоимости покупки (Cost Basis)

Cost basis помогает понять исходную цену покупки инвестиций — это важно для расчета доходности и налогового учета.

#### Источники стоимости покупки

Sure берет cost basis из трех источников:

| Источник | Описание |
| --- | --- |
| **Вручную** | Значения, которые вы вводите сами |
| **Расчет** | Вычисляется на основе сделок покупки и истории транзакций |
| **Провайдер** | Импортируется от финансового учреждения (Plaid, SimpleFIN и т.д.) |

#### Приоритет источников

Если данные приходят из нескольких источников, используется такой приоритет:

**Вручную > Расчет > Провайдер**

То есть:
- Вручную введенные значения всегда имеют приоритет
- Расчетные значения перекрывают данные провайдера
- Данные провайдера используются, если других нет

#### Защита ручных значений

Если вы вручную задаете cost basis, Sure автоматически блокирует его от автоматических обновлений, чтобы ваши значения не затирались при синхронизации.

#### Как задать cost basis вручную

Сделать это можно двумя способами:

**Из списка активов (Holdings):**

1. Откройте инвестиционный счет
2. Найдите нужный актив
3. Нажмите на иконку карандаша рядом со средней ценой
4. Введите:
   - **Общую стоимость**: сколько вы заплатили за все акции
   - **Стоимость за акцию**: средняя цена одной акции
5. Форма автоматически пересчитает значения между форматами
6. Нажмите **Save**
